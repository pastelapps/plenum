## SCHEMA ATUAL (fonte da verdade)

> Sempre ler `context/SCHEMA.md` antes de qualquer modificação.

### Entidades e relacionamentos:

```
Professor (1) ──────────── (N) Course
Course    (1) ──────────── (N) Turma
Course    (1) ──────────── (N) Lead
Course    (1) ──────────── (N) BlogPost
Turma     (1) ──────────── (N) Lead
Turma     (1) ──────────── (N) PageView
Turma     (1) ──────────── (N) FolderView
Turma     (1) ──────────── (N) FolderDownload
Event     (1) ──────────── (N) Lead
```

### Regras críticas de integridade:
- `Course.slug` — único, gerado a partir do título (kebab-case)
- `Turma` nunca existe sem `Course` (onDelete: Cascade)
- `Lead` sempre tem `source` preenchido
- Tracking (PageView, FolderView, FolderDownload) sempre vinculado à `Turma`, não ao `Course`

## SEED DE DADOS (prisma/seed.ts)

O seed deve criar dados realistas suficientes para desenvolver e testar todas as telas:

```
2 professores com foto (usar placeholder)

4 cursos (variando categoria, público, carga horária)
Para cada curso: 3 turmas em datas diferentes (jan, mar, mai 2026)
3 eventos (seminário, congresso, imersão)
5 posts de blog (2 publicados, 1 agendado, 2 rascunho)
Prova social: 4 contadores, 6 logos, 4 depoimentos, 3 autoridades
6 posts Instagram (curadoria manual)
50 leads distribuídos entre cursos/turmas
1 admin user (email: admin@plenum.com.br / senha: Admin@2026)
SiteSettings com dados básicos preenchidos
```

## PROTOCOLO DE MUDANÇA NO SCHEMA

Qualquer alteração no schema DEVE seguir este protocolo:

1. Receber BRIEFING do MASTER com justificativa
2. Verificar impacto nas queries existentes
3. Verificar impacto nos agentes BACKEND e FRONTEND
4. Criar migration com nome descritivo
5. Atualizar `context/SCHEMA.md`
6. Reportar ao MASTER quais agentes precisam ser notificados

## CHECKLIST DE ENTREGA (usar em cada REPORT)

```
- [ ] Schema válido (npx prisma validate)

- [ ] Seed executado sem erros (npx prisma db seed)
- [ ] Prisma client gerado (npx prisma generate)
- [ ] Queries tipadas (sem any)
- [ ] Queries com tratamento de not found
- [ ] context/SCHEMA.md atualizado
```

## COMANDOS ÚTEIS

```bash
# Criar e aplicar migration
npx prisma migrate dev --name nome-descritivo

# Resetar banco e rodar seed
npx prisma migrate reset

# Gerar client após mudança no schema
npx prisma generate

# Abrir Prisma Studio (visualizar dados)
npx prisma studio

# Verificar schema sem migrar
npx prisma validate
```
